# Guide Reference

Describe the content that an effective beginner-friendly guide should contain.

Before writing a guide, you MUST know:

- the topic the user wants to learn
- the user's likely starting level
- the minimum context needed to avoid confusion or unsafe usage

## Iron Laws

- MUST ensure understanding and action support each other throughout the guide
- MUST balance explanation, actionable steps, and examples in every guide
- ALWAYS write in a clear, well-sequenced, beginner-friendly way; beginner-friendly NEVER means vague or oversimplified
- NEVER turn the guide into a reference dump without a teaching flow
- NEVER assume background knowledge beyond what has been explicitly introduced
- NEVER list steps without explaining why they matter or when to use them
- ALWAYS include common pitfalls, limitations, and safety notes relevant to beginners
- MUST use web search to find up-to-date and relevant docs, examples, and tutorials to inform guide content

## Guide Structure

Cover the following sections, including optional sections when relevant.

- Overview: what the topic is and why it matters
- Audience `[Optional]`: who this guide is for and not for
- Estimated Time `[Optional]`: how long it should take to read and follow the guide
- Prerequisites: what users need before they begin
- Getting Started: the first useful steps to take
- Core Concepts: the ideas a user must understand to avoid shallow usage
- Architectural Overview: the high-level parts or layers involved
- Work Model: how the topic is usually applied in practice
- Real World Use Cases: concrete examples or scenarios
- Security: the main safety, trust, or abuse considerations
- Best Practices: the most useful advice for getting started well and avoiding common mistakes
- Verification `[Optional]`: how users can check they are on the right track
- Troubleshooting `[Optional]`: common issues and how to resolve them
- Next Steps: where users should go after the guide
- Related References `[Optional]`: links to deeper guides, specs, or references for related topics

## Next Steps

After writing the guide, check whether the topic also needs a quickstart, README link, or best-practices document for adjacent use cases.
