# Guide

Provide a clear, beginner-friendly guide for a topic the user wants to learn.

The guide should cover:

1. Overview — what this topic is and why it is useful
2. Prerequisites — what users need to know or have before learning this topic
3. Getting Started — what are the first steps to take to start using or learning about this topic
4. Core Concepts — what are the most important things to understand about this topic
5. Architectural Overview — how components of the topic fit together at a high level
6. Work Model — how the topic is typically used or applied in practice
7. Real World Use Cases — examples of how this topic is used in real life
8. Security — what are the key security considerations or best practices related to this topic
9. Common Pitfalls — what are common mistakes or misconceptions to avoid when learning about this topic
10. Tips and Best Practices — what are the most important tips for beginners to know when getting started with this topic
11. Next Steps — what should users do after reading this guide to continue learning or using this topic effectively

Add additional relevant sections when they meaningfully improve understanding.

You must:

- Keep the guide concise and beginner-friendly
- Use clear, simple language and avoid jargon
- Keep warm and encouraging tone throughout the guide, never cold or clinical
- Use emojis sparingly to improve readability and visual clarity, without overusing them
- Add colored Mermaid diagrams or other visual aids when they help explain complex concepts more clearly. When useful, add numbered steps or sequence markers to make flows easier to understand
- Avoid Markdown-sensitive syntax in Mermaid diagrams; use plain text labels such as "1: run task" instead of "1. run task"

Save the guide as `<topic>-guide.md` in the appropriate directory, using a well-structured format with clear headings and sections for each relevant point above.
